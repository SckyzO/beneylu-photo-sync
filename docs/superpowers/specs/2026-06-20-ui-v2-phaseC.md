# UI v2 — Phase C (polish + search + pagination bug)

> Scope captured 2026-06-20 from user feedback after the Playwright audit
> (`docs/audits/2026-06-20-ui-v2-playwright-audit.md`). Builds on `feat/ui-v2`.
> Status: requirements captured, NOT yet planned/approved.

## C0. Core bug — cards pagination (data completeness) — **highest priority**

`BeneyluClient.cards(board_id)` (`client.py:58-61`) does a single
`GET /api/cardboard/boards/{board_id}/cards` and treats the body as the full list.
The Beneylu SPA shows a **"charger les cartes précédentes"** control at the bottom of
a board → the endpoint is **paginated**; we only ingest the most recent page, so older
cards and their photos are never synced.

**Blocked on discovery:** the pagination mechanism (query param / cursor / page size /
envelope shape) is not visible in our code. Options to resolve:
1. User inspects the "charger les cartes précédentes" request in browser devtools on
   `ent-ecole.fr` and pastes method+URL+params+response shape (no credentials needed).
2. Fresh, explicit authorization for a short read-only API recon.

**Fix shape (once known):** make `cards()` loop pages until exhausted (cursor/`before`
timestamp or `page`/`offset`), preserving per-item resilience; add tests with a
mocked two-page response asserting all cards are returned. Keep idempotency (state on
`media_id`) so a re-sync only downloads the newly discovered older photos.

## C1. Search bar

Client-side filter over the rendered gallery: an input in the toolbar (or header) that
filters visible sections/photos by text match on board / month / section title (and
optionally photo filename). Pure vanilla JS, no dependency; instant filter, clears on
empty. Empty-result state message.

## C2. Header redesign

Current header (`📷-style SVG + "Beneylu Photo Sync" + "Galerie / Configuration"`) reads
as cluttered/unpolished. Redesign cleanly within the cosmos system:
- Clear visual hierarchy: brand left, primary nav, theme toggle right.
- Consider: nav as pill/segmented control or underline-active state; refined spacing;
  the search bar (C1) integrated tastefully; active-route indication.
- Keep it light, not heavy. Mobile: brand must not wrap awkwardly to two lines.

## C3. Section box redesign (title prominence)

Give each section a clear header bar, title **larger** and prominent, count beside it,
and a download affordance as an **icon button with a tooltip** ("Télécharger la
sélection"). Sketch:

```
┌───────────────────────────────────────────────────────────┐
│  Écrit  (13)                                   [⬇ tooltip]  │
├───────────────────────────────────────────────────────────┤
│  [photo] [photo] [photo] [photo] [photo]                    │
│  [photo] ...                                                │
└───────────────────────────────────────────────────────────┘
```

- Title font noticeably bigger (e.g. `text-base`/`text-lg` semibold vs current `text-sm`).
- Count as a subtle badge next to the title.
- Download = icon-only button, top-right, with an accessible tooltip (title/aria-label)
  explaining it downloads the whole section.
- A divider under the header bar, then the photo grid.
- Also addresses audit #2 (sparse sections): denser/auto-fill grid so 1–4 photo
  sections read as intentional.

## C4. Sticky month/date overlay redesign

The sticky month header exists and is useful but is poorly designed (audit #4: 1px
sliver at `top-[68px]`, faint blur, low contrast). Redesign into a clean, legible
date chip/divider: stronger background, correct offset, better typography (e.g.
uppercase tracking or a pill), clearly distinct from section headers.

## C5. (carry-over from audit) lightbox polish

Overlay opacity `/90–/95` (stop header bleed-through), optional caption (`data-name`)
and `n / total` counter. Lower priority than C0–C4.

## Constraints (unchanged)

Server-rendered Jinja + vanilla JS, no SPA/framework, no Node at runtime. Tailwind v4
precompiled (`make css` after class changes). TDD; `make check` green. Each change
validated by a fresh Playwright pass against the demo container. Credentials never
logged/committed; no real-account auth without fresh explicit authorization.
