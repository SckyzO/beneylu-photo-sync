# UI v2 — "Beneylu Photo Sync" branding, data fixes, gallery polish

> Design doc. Status: approved 2026-06-20. Plan: `docs/superpowers/plans/2026-06-20-ui-v2.md` (next).
> Builds on the cosmos UI redesign (merged 2026-06-19).

## Goal

Evolve the server-rendered web UI: rename to "Beneylu Photo Sync", fix two data
problems (ugly section titles, scattered month grouping), add a board filter, a
gallery redesign with a lightbox, and zip downloads. No SPA — the existing
FastAPI + Jinja2 + vanilla-JS stack is kept.

## Decisions (locked)

- Display name: **Beneylu Photo Sync** (code/package names unchanged).
- Month grouping basis: **card publication date** (`card.created_at`), not EXIF.
- Existing 174 photos: **clean re-sync** from an empty data dir → no migration code.
- Board filter applied **at sync** (excluded boards never downloaded).
- Architecture unchanged: server-rendered, no SPA, no JS framework, no Node at runtime.

## Non-goals (YAGNI)

- No SPA / React. No Alpine. No new runtime dependency for the lightbox or zip
  (vanilla JS + Python stdlib `zipfile`).
- No on-disk migration/reorg of old folders (user re-syncs clean).
- No per-photo selection/multi-select download (only: all / per-month / per-section).

## Phasing

One spec, plan in two phases:

- **Phase A — foundations:** branding (name, logo, toggle icon), data fixes
  (month=createdAt, section=first line), board filter at sync, config page
  centered + redesigned with the exclude field.
- **Phase B — polish:** gallery visual redesign, lightbox, zip downloads.

### A2. Section title = first line (#6)

`naming.section_folder(description)` currently mashes the whole multi-line
description into 60 chars. New behavior: keep only the **title (first line)**.

```python
_BULLET = re.compile(r"\s[-•]\s")

def section_folder(description: str | None) -> str:
    if not description:
        return SECTION_FALLBACK
    # Cards store a short title then a bullet list of activities. Keep the title:
    # the text before the first newline, or before the first inline bullet.
    first = description.replace("\r", "\n").split("\n", 1)[0]
    first = _BULLET.split(first, maxsplit=1)[0]
    text = _WHITESPACE.sub(" ", first).strip()
    text = _UNSAFE.sub("_", text)
    text = text[:SECTION_MAXLEN].rstrip(" .")
    return text or SECTION_FALLBACK
```

`"Écrit\n -Graphisme: échelles…\n -Écrire son prénom"` → `"Écrit"`.
`"Explorer le monde -Entraînement au découpage -…"` → `"Explorer le monde"`.

### A4. Board filter at sync (#3)

Exclude boards by name so they are never downloaded.

- `CardboardSource.__init__(self, excluded_boards: list[str] | None = None)` stores
  a lowercased list. In `iter_items`, skip a board when any excluded term is a
  case-insensitive substring of `board.name`:

```python
class CardboardSource(Source):
    name = "cardboard"

    def __init__(self, excluded_boards: list[str] | None = None):
        self._excluded = [b.casefold() for b in (excluded_boards or []) if b.strip()]

    def _is_excluded(self, board_name: str) -> bool:
        name = board_name.casefold()
        return any(term in name for term in self._excluded)

    def iter_items(self, client):
        for board in client.boards():
            if self._is_excluded(board.name):
                continue
            for card in client.cards(board.id):
                if card.type != "image":
                    continue
                for att in card.attachments:
                    yield MediaItem(media_id=att.media_id, attachment=att,
                                    board=board, card=card)
```

- **Config plumbing.** Add `excluded_boards: list[str]` to both config sources:
  - `config.py Settings`: `excluded_boards: list[str] = []` (env `ENT_EXCLUDED_BOARDS`,
    comma-separated; pydantic-settings parses it).
  - `web/settings_store.py WebConfig`: new field `excluded_boards: list[str]`;
    `save(..., excluded_boards: list[str] | None = None)` persists it (stored as a
    JSON list); `effective()` reads env `ENT_EXCLUDED_BOARDS` (comma-split) else
    the file list else `[]`.
  - `web/app.py _default_job`: `CardboardSource(excluded_boards=cfg.excluded_boards)`.
  - `cli.py` sync path: pass `Settings().excluded_boards` to `CardboardSource`.

## Phase B

### B1. Gallery visual redesign (#4)

`gallery.html` refinements (utility-class only, no new CSS file):

- Month headers become light **sticky** sub-headers (`sticky top-[64px]` under the
  fixed header) with a subtle divider.
- Section cards: more breathing room (`p-5`, `gap-4`), section title with a small
  count badge (`{{ section.photos|length }}`).
- Thumbnail grid: denser and regular (`grid-cols-2 sm:grid-cols-3 lg:grid-cols-5`),
  square tiles (`aspect-square object-cover`), gentle hover (`transition
  hover:opacity-90 hover:ring-2 hover:ring-brand-500/40`).
- Toolbar gets the global **"Tout télécharger"** button next to "Synchroniser".

### B2. Lightbox (#8)

Clicking a photo opens an in-page overlay instead of a new tab. Pure vanilla JS in
`app.js`, no dependency.

- Markup: each thumbnail is an `<a href="/photo/{{key}}" class="js-photo"
  data-full="/photo/{{key}}" data-name="{{name}}">` inside a section container
  marked `data-lightbox-group` so JS can collect siblings for prev/next.
- JS builds a single reusable overlay (`<div id="lightbox">` with `<img>`, ✕, ‹ ›).
  On click: `preventDefault`, open overlay at the clicked index within its group.
  Controls: ← / → navigate within the group, Esc / click-outside / ✕ close.
  Graceful: if JS disabled, the `href` still opens the photo (progressive
  enhancement).

### B3. Zip downloads (#5)

New routes in `app.py`, behind the same `guard` auth dependency:

- `GET /download` → zip of the whole library (all boards, excluding `THUMB_DIR`).
- `GET /download/{key:path}` → zip of a sub-tree: a board (`PS`), a month
  (`PS/2026-06`), or a section (`PS/2026-06/Écrit`) directory.

Implementation:

- A `web/archive.py` helper: `build_zip(root: Path, subdir: Path) -> Path` writes a
  `tempfile.NamedTemporaryFile(suffix=".zip", delete=False)` using stdlib
  `zipfile.ZipFile` (ZIP_DEFLATED), walking `subdir` for files whose suffix is in
  `gallery.IMAGE_EXTS`, skipping `THUMB_DIR`, storing arcnames relative to `root`.
  Returns the temp path.
- Route resolves `key` to a **directory** safely (reuse a directory-aware variant
  of `safe_resolve`: resolve under data root, must be a dir under root, reject
  traversal and `THUMB_DIR`); 404 otherwise. Empty key = data root.
- Response: `FileResponse(zip_path, media_type="application/zip",
  filename="<name>.zip", background=BackgroundTask(os.unlink, zip_path))` so the
  temp file is deleted after the response is sent.
- UI: "Tout télécharger" in the toolbar (→ `/download`); a small **"⬇ Télécharger
  la section"** link per section card (→ `/download/{board}/{month}/{section}`);
  optional per-month link on the month header (→ `/download/{board}/{month}`).

## Error handling

- Per-item sync isolation unchanged.
- Zip of a missing/invalid/traversal directory → 404 (never 500). A directory that
  exists but contains no images → a valid empty zip (200).
- Lightbox is progressive enhancement; failure falls back to the plain photo link.
